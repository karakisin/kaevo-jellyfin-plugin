# iOS Phase 1 Scope

## In Scope

- Replace the default SwiftData template with a Kaevo app shell.
- Create iOS source folders for API, Brand, Configuration, DesignSystem, Diagnostics, Features, Models, Playback, and Services.
- Add a backend URL setting using BackendConfiguration.
- Add placeholder feature screens for Home, Discover, Search, Library, Downloads, Profiles, and Settings.
- Add a typed API shell and error type matching the tvOS transport pattern.
- Add minimal models and playback types required for early compileable structure.

## Out Of Scope

- Full home feed implementation.
- Real image loading and caching.
- AVPlayer playback presentation.
- Authentication.
- Offline storage.
- tvOS sidebar, focus, Top Shelf, or remote-control behaviors.
- Moving or modifying the tvOS project.

## Completion Criteria

- iOS project has the approved folder structure.
- The starter template no longer owns the app experience.
- The app builds or has a concrete build blocker recorded.
- AI reference docs describe the structure and next work.
