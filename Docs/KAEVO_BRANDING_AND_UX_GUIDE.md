# Kaevo Branding and UX Guide

## Product feel

Kaevo should feel premium, calm, minimal, polished, and distinctly at home on
Apple platforms. Every screen should be understandable by a normal household
member without server knowledge.

## UX rules

- Show one clear primary action per step.
- Remove duplicate buttons and repeated explanations.
- Use sensible defaults and reveal advanced details only when needed.
- Use dynamic wording that changes from action to confirmation.
- Prefer short inline guidance over technical alerts.
- Keep developer diagnostics separate from normal settings.

## Preferred wording

- **Connect Jellyfin**
- **Install Kaevo Jellyfin Plugin**
- **Kaevo Jellyfin Plugin Installed**
- **Start Cloud Trial**
- **Remote Access Ready**
- **Try Again**

## Avoid in normal UI

- worker
- heartbeat
- connector
- queue
- AWS
- tunnel
- proxy
- API key
- relay configuration

These terms may appear in developer diagnostics and engineering documents when
they are necessary and clearly scoped.

## Setup voice

- Say what the user can do, not how the infrastructure works.
- Explain failures with a useful next action.
- Do not show install instructions after installation succeeds.
- Do not claim remote access is ready until it has been verified.
